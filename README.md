# 📑 DebtLedger

**DebtLedger** — це консольна система на C++ для керування списком покупок та автоматичного розподілу спільних витрат. Проект створений для демонстрації глибокого розуміння ООП та шаблонного програмування.

---

## 🎯 Основні концепції (OOP Focus)

Проект реалізує наступні архітектурні рішення:
*   **Encapsulation:** Приховання логіки розрахунків всередині класів товарів.
*   **Inheritance Hierarchy:** Базовий абстрактний клас `Product` -> `PlannedProduct` -> `BoughtProduct`.
*   **Polymorphism:** Динамічний розрахунок частки чека для кожного учасника через віртуальні методи.
*   **Templates:** Універсальний менеджер списків `ListManager<T>`, що працює з будь-яким типом об'єктів.

---

## 🛠 Технічне завдання (Specification)

### 1. Модель даних
Система має розрізняти два типи витрат у чеку:
- **Individual:** Товар, який купується для однієї особи.
- **Shared:** Товар, вартість якого ділиться порівну між групою людей.

### 2. Функціонал
- Створення та редагування списку майбутніх покупок.
- Конвертація "запланованого" товару в "куплений" з додаванням ціни та учасників.
- Генерація звіту: вивід загальної суми витрат та деталізація боргів по іменах.
- Використання **Smart Pointers** (`std::unique_ptr` або `std::shared_ptr`) для безпечного керування пам'яттю.
